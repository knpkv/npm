# Milestone 1 — Repository and rly

Goal: establish enforceable TSX/package gates and publishable `@knpkv/rly` before Control Center imports it.

## R01 — Teach root tooling about React sources

- **Scope:** extend ESLint to `.tsx` and establish browser-test script conventions without changing existing test concurrency. Package-specific TypeScript references land with the package that owns them.
- **Tests:** prove an invalid TSX fixture is discovered; run existing repository lint/check discovery.
- **Review focus:** no broad ignore, no vendored subtree inclusion, no silently dropped existing test.

## R02 — Scaffold the publishable rly package

- **Scope:** add the `0.0.0` ESM package/Vite/TypeScript/Vitest configuration, explicit initial exports, React peers, direct dependencies, CSS-only `sideEffects`, README, lockfile/root references, and one initial minor changeset targeting the first `0.1.0` release.
- **Tests:** build/import initial entries; assert Node/vendor/Control Center dependencies are absent.
- **Depends on:** R01.
- **Review focus:** correct dependency classification, browser-safe boundary, stable `@knpkv/rly` naming.

## R03 — Generate the package contract from one manifest

- **Scope:** add typed `component-manifest.ts`, generation of barrels/Vite/package export fragments, post-build validation, deep-import rejection, packed strict consumer, and manifest/filesystem drift checks.
- **Tests:** build, pack, install, resolve every declared artifact, fail if `dist` or an export is absent.
- **Depends on:** R02.
- **Review focus:** deterministic generation; no handwritten duplicate public lists or skipped absence test.

## R04 — Establish the bounded catalog harness

- **Scope:** add Storybook 10.5/Vite as rly's static interactive catalog, autogenerated documentation, DOM/a11y setup, theme/forced-color/reduced-motion/viewport/locale/density controls, one-worker interaction/visual configuration, deterministic teardown, and fail-safe changed-file classifier.
- **Tests:** empty/foundation static catalog build, autogenerated-doc navigation, harness self-tests, one Chromium process exits cleanly.
- **Depends on:** R02–R03.
- **Review focus:** catalog is not a second production app; no remote or unbounded browser worker.

## R05 — Generate semantic tokens and themes

- **Scope:** implement typed color/type/space/shape/motion sources, deterministic CSS/registry metadata generation, `light-dark()` themes, forced colors, Geist loading, reset/base layers, contrast checks, and no-raw-color lint.
- **Tests:** golden/drift, contrast, light/dark/system/forced-color DOM, raw component color rejection.
- **Depends on:** R03–R04.
- **Review focus:** provenance colors never mean readiness; components consume semantic tokens only.

## R06 — Add framework-neutral foundations

- **Scope:** add `ThemeProvider`, `GlobalStyles`, `Icon`, `LinkProvider`, and `PortalProvider`; wrap Radix behavior without leaking its types.
- **Tests:** Node import/SSR safety for pure entries, router bridge, custom portal, controlled theme, focus restore.
- **Depends on:** R05.
- **Review focus:** no router/`document.body` assumption; no package-wide `"use client"`.

## R07 — Add non-overlay primitives

- **Scope:** implement `Text`, `Surface`, `Divider`, `Button`, `IconButton`, `StateLabel`, `Avatar`, `Skeleton`, and `StatePanel` with machine-readable variants.
- **Tests:** DOM/a11y/state/long-content matrix, icon-label type fixture, focus, disabled/loading geometry.
- **Depends on:** R06.
- **Review focus:** discriminated accessibility props, meaningful variants, no domain semantics.

## R08 — Add controlled navigation and form primitives

- **Scope:** implement `Tabs`, `Field`, and `Select` with controlled-first APIs, labels/descriptions/errors, roving focus, and internal Radix composition.
- **Tests:** arrow keys, controlled/default behavior, labeling/error announcement, Storybook interactions.
- **Depends on:** R07.
- **Review focus:** no public Radix type; optional values cannot break accessibility.

## R09 — Add Dialog and Sheet

- **Scope:** implement real compound overlay APIs, inert background, focus set/contain/restore, dismissal, scroll lock, compact full-screen mode, and reduced-motion fallback.
- **Tests:** nested portal/focus/unmount, Escape, pointer dismissal, mobile/zoom, open/focus visual canaries.
- **Depends on:** R06–R08.
- **Review focus:** cleanup and focus behavior remain deterministic across nested overlays.

## R10 — Add provenance and collaborator patterns

- **Scope:** implement `ServiceMark`, `FreshnessStamp`, `EvidenceStamp`, `Person`, `PeopleStrip`, and `CollaboratorGroup` with all five services and explicit roles.
- **Tests:** safe avatar fallback, named people at compact widths, overflow semantics, every freshness/service state.
- **Depends on:** R07.
- **Review focus:** no anonymous avatar stack; compact IDs never replace full accessible provider names.

## R11 — Add Release Relay and verdict identity

- **Scope:** implement 16 code-owned SVG paths, compact/hero `ReleaseRelay`, and `Verdict`; accept only derived codename/algorithm/three indices.
- **Tests:** every glyph, exactly three distinct tiles, accessible name, forced colors, golden presentation vectors.
- **Depends on:** R05–R07.
- **Review focus:** rly performs no hashing/domain derivation; no emoji/font-dependent identity.

## R12 — Add delivery stages and relationships

- **Scope:** implement `StageRail`, `RelationshipChain`, and equivalent `RelationshipTable` for every relationship lifecycle and arbitrary cardinality.
- **Tests:** zero/one/six/twenty fixtures, chain/table semantic equivalence, keyboard, color-independent state.
- **Depends on:** R07, R10.
- **Review focus:** many-to-many and missing relationships remain explicit rather than hidden spatially.

## R13 — Add release, entity, and activity patterns

- **Scope:** implement `ReleaseRow`, `ReleasePreview`, `WorksetCard`, `EntityShell`, `TimelineRow`, and `EntityTable` with complete data states and controlled callbacks.
- **Tests:** six release states, Jira/PR/pipeline dimensions, preview focus/reflow, responsive tables, all empty/stale/partial/error states.
- **Depends on:** R09–R12.
- **Review focus:** presentation-only props, arbitrary cardinality, no monolithic page component.

## R14 — Add agent and governed-action patterns

- **Scope:** implement `AgentContextButton/Drawer/Thread/Job/Proposal` and `GovernedActionReview` with exact context/evidence/capability/progress/target/revision/impact/outcome slots.
- **Tests:** context-before-composer, no focus theft, cancel/terminal states, human-circle vs agent-square semantics, confirmation invariant.
- **Depends on:** R09–R10, R13.
- **Review focus:** agent proposal never appears as human authorization; components do not execute actions.

## R15 — Integrate the complete pinned diff workbench

- **Scope:** add diff presentation types/states/tree/header/findings and pinned `@pierre/diffs` React `CodeView`, worker provider/factory, themes, split/stacked/wrap/context/selection/annotations, virtualization, synchronous fallback, and isolated `./diff` chunks.
- **Tests:** every file state, 500-file complete inventory, item versioning/scroll, selection/anchor, worker failure/cleanup, theme/a11y/visual and packed-consumer contracts.
- **Depends on:** R05–R10.
- **Review focus:** no silent omission, experimental API isolated, normal primitives do not load renderer/worker code.

## R16 — Publish the fail-closed agent registry and scaffolder

- **Scope:** generate `components.json`, `schema.json`, `USAGE.md`, search metadata and variants from manifest/types/stories; export/package all registry artifacts; add component scaffolder and no-app-import/variant-metadata rules; finalize README/changeset, publish the static Storybook output, and add the indexed `packages/docs` rly page with install/exports/tokens/themes/catalog/registry/examples plus a direct catalog link.
- **Tests:** every manifest entry has code/style/story/test/docs/a11y metadata and at least one navigable story covering its declared variants and meaningful states; pack/install strict consumer; resolve/read/schema-validate registry; Storybook build/navigation/interaction/a11y/full high-risk one-worker visual matrix; docs page catalog link and examples match packed exports.
- **Depends on:** R03–R15.
- **Review focus:** extraction failure fails CI; registry is documentation/tooling, never a runtime JSON-to-React executor.

## Exit gate

Pack `@knpkv/rly`, install it into a clean strict TypeScript fixture, render light/dark release/entity/agent/diff patterns, validate the registry, and prove no Control Center/vendor/Node dependency enters its graph. Control Center consumption begins only after this gate passes.
