---
title: Claude Adapter
description: Run Effect AI language-model programs through the local Claude CLI.
---

`@knpkv/ai-claude` adapts an installed, authenticated Claude CLI to Effect AI's `LanguageModel` service. It starts a local child process for each request and exposes one public constructor: `model(options)`.

## Install

```bash
pnpm add @knpkv/ai-claude effect @effect/platform-node
```

Install and authenticate the `claude` CLI separately. The package does not accept, copy, or persist Claude credentials.

## Public API

```typescript
model({
  cwd,
  executable?,
  model?,
  access?,
  timeout?,
  maxOutputBytes?,
  maxStderrBytes?
})
```

| Option           | Required | Meaning                                                                  |
| ---------------- | -------- | ------------------------------------------------------------------------ |
| `cwd`            | Yes      | Working directory used for the Claude process.                           |
| `executable`     | No       | Claude executable name or path. Defaults to `claude`.                    |
| `model`          | No       | Claude model override. Otherwise the CLI chooses its configured default. |
| `access`         | No       | `read-only` or `workspace-write`. Defaults to `read-only`.               |
| `timeout`        | No       | Maximum lifetime of one local process.                                   |
| `maxOutputBytes` | No       | Maximum accepted stdout before the request fails.                        |
| `maxStderrBytes` | No       | Maximum retained stderr used for a bounded failure description.          |

Keep `access: "read-only"` for review, explanation, and planning. Opt into `workspace-write` only when the calling workflow is prepared to validate changes made under `cwd`.

## Generate Text

```typescript
import { NodeServices } from "@effect/platform-node"
import { model } from "@knpkv/ai-claude"
import { Effect } from "effect"
import { LanguageModel } from "effect/unstable/ai"

const program = LanguageModel.generateText({
  prompt: "Explain the current change and identify one missing test."
}).pipe(Effect.provide(model({ cwd: "/workspace/project" })), Effect.provide(NodeServices.layer))
```

The returned model can also provide `LanguageModel` to `Chat` and other Effect AI programs. Conversation history is rendered into each local CLI request; the package does not expose Claude session management.

## Structured Output

`LanguageModel.generateObject` is supported. The adapter converts the Effect `Schema` to JSON Schema, supplies it to Claude, and lets Effect decode the final value.

```typescript
import { Schema } from "effect"
import { LanguageModel } from "effect/unstable/ai"

const Check = Schema.Struct({
  summary: Schema.String,
  ready: Schema.Boolean
})

const check = LanguageModel.generateObject({
  objectName: "check",
  prompt: "Check whether this change is ready for review.",
  schema: Check
})
```

Schema support is limited to shapes that both Effect's JSON Schema conversion and the installed Claude CLI can represent.

## Streaming

`LanguageModel.streamText` is a buffered compatibility stream. The adapter waits for Claude's single JSON result, then emits the completed text as one delta. Consumers must not use it for live token progress or depend on token boundaries.

## Execution and Cancellation

The adapter sends prompts through stdin so prompt text is not placed in the process argument list. It drains stdout and stderr concurrently, validates JSON events, and enforces the configured timeout and byte bounds. Interrupting the Effect closes the process scope and terminates the Claude process, including its process group where the runtime supports group supervision.

Spawn failures, invalid events, non-zero exits, timeouts, and output-limit failures are returned as typed Effect AI errors. Retained stderr is bounded and must not be treated as a complete transcript.

## Compatibility Limits

- Effect AI toolkits are rejected before Claude starts. Native Claude tools cannot safely call arbitrary Effect handlers in the parent runtime.
- File prompt parts are rejected. Put relevant text in the prompt or let Claude inspect files permitted by `cwd` and `access`.
- The adapter does not promise compatibility with undocumented Claude CLI event fields.
- Read-only access limits workspace mutation; it is not a substitute for isolating an untrusted repository or executable.
