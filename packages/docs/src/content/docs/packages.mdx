## API Clients

The API clients expose Effect services around generated OpenAPI clients. They keep generated schemas available through subpath exports and provide `Context.Service` layers for application code.

| Package                        | Public exports                                                                       | Auth model                                                           |
| ------------------------------ | ------------------------------------------------------------------------------------ | -------------------------------------------------------------------- |
| `@knpkv/jira-api-client`       | `JiraApiClient`, `JiraApiConfig`, and the generated `JiraApi` namespace.             | Basic auth or OAuth2 through Atlassian cloud gateway with `cloudId`. |
| `@knpkv/confluence-api-client` | `ConfluenceApiClient`, `ConfluenceApiConfig`, and generated v1/v2 namespaces.        | Basic auth or OAuth2 through Atlassian cloud gateway with `cloudId`. |
| `@knpkv/clockify-api-client`   | `ClockifyApiClient`, `ClockifyApiConfig`, and the generated `ClockifyApi` namespace. | `X-Api-Key` header.                                                  |

Minimal service shape:

```typescript
import { Effect, Layer } from "effect"
import * as Redacted from "effect/Redacted"
import * as NodeHttpClient from "@effect/platform-node/NodeHttpClient"
import { JiraApiClient, JiraApiConfig } from "@knpkv/jira-api-client"

const program = Effect.gen(function* () {
  const jira = yield* JiraApiClient
  return yield* jira.getCurrentUser(undefined)
})

const ConfigLive = Layer.succeed(JiraApiConfig, {
  baseUrl: "https://example.atlassian.net",
  auth: {
    type: "basic",
    email: "you@example.com",
    apiToken: Redacted.make("...")
  }
})

Effect.runPromise(
  program.pipe(
    Effect.provide(JiraApiClient.layer),
    Effect.provide(ConfigLive),
    Effect.provide(NodeHttpClient.layerFetch)
  )
)
```

## Local AI Adapters

These packages expose local coding-agent CLIs through Effect and provide a shared protocol for durable application workers. Authentication stays with the installed CLI.

| Package             | Public API                                   | Runtime                                              | Docs                          |
| ------------------- | -------------------------------------------- | ---------------------------------------------------- | ----------------------------- |
| `@knpkv/ai-runtime` | `AgentRuntime`, `layerAgentRuntime(adapter)` | Provider-neutral bounded event protocol.             | [Agent Runtime](/ai-runtime/) |
| `@knpkv/ai-codex`   | `model(options)`, `streamEvents(options)`    | Local `codex` process; read-only access by default.  | [Codex Adapter](/ai-codex/)   |
| `@knpkv/ai-claude`  | `model(options)`                             | Local `claude` process; read-only access by default. | [Claude Adapter](/ai-claude/) |

## Local State By Package

| Package                             | Local state                                                                                                  |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------------ |

| `@knpkv/jira-clockify`              | `~/.jcf/config.json`, `~/.jcf/clockify.json`, and timer state.                                               |
| `@knpkv/agent-skills`               | Codex and Claude skill directories.                                                                          |
| `@knpkv/ai-codex`                   | Uses the Codex CLI's existing local authentication and configuration; the adapter owns no credential store.  |
| `@knpkv/ai-claude`                  | Uses the Claude CLI's existing local authentication and configuration; the adapter owns no credential store. |
| `@knpkv/ai-runtime`                 | Owns no state; applications persist jobs and opaque server-side continuation references.                     |
