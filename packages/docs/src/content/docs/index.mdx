The `@knpkv` package collection is a workspace of focused command-line tools and Effect-based TypeScript libraries for developer operations:

- Jira issue and release-version export.
- Confluence page synchronization between Atlassian Cloud and local Markdown.
- AWS CodeCommit pull request browsing, export, and update workflows.
- Jira-backed Clockify timers and reconciliation.
- Unified Atlassian OAuth profile diagnostics and repair.
- Bundled Codex and Claude agent skills for the CLIs.

The docs are organized around the real command surface exposed by each package. Start with the guide if you are choosing a tool, then use the product pages as command references.

## Repository Shape

The monorepo is split into product CLIs, reusable clients, shared Atlassian helpers, and private documentation:

```text
packages/
  jira-cli/                 jira binary
  confluence-to-markdown/   confluence binary
  codecommit/               codecommit binary and TUI
  jira-clockify/            jcf binary and TUI
  jira-api-client/          Effect Jira Cloud REST client
  confluence-api-client/    Effect Confluence Cloud REST client
  clockify-api-client/      Effect Clockify REST client
  atlassian-common/         OAuth, profiles, config, AST, serializers
  agent-skills/             shared skill installer
  docs/                     this Starlight site
```

## Common Runtime Model

Most packages use Effect services at their boundaries:

- CLI commands are built with `effect/unstable/cli`.
- Remote API clients expose `Context.Service` layers.
- Errors are tagged so callers can recover by tag instead of parsing strings.
- Commands that support `--json` keep machine-readable stdout clean.

See [Packages](/packages/) for library exports and [Conventions](/conventions/) for command behavior.
